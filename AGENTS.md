# AGENTS.md

## Project

accroitre — Accroître is a high-speed, cross-platform file copier written in Rust. It reads
files in physical disk order to eliminate random seeks on HDDs, deduplicates
content via xxHash-128 or BLAKE3 (copies once, hard-links the rest), streams
small files through batched tar pipes, and transfers over SSH without SFTP.
Designed to be hands-down better than rsync, scp, fast-copy, and anything else
for local copies, USB/NAS backups, and remote SSH transfers.

Architecture: collapsed hexagonal (DDD-lite). Two workspace crates:

- accroitre       — library crate (published to crates.io), contains domain
                      model, port traits, and adapter implementations
- accroitre-cli   — binary crate (`accro` command), rich TUI with braille
                      spinners and progress bars

## Setup commands

- Build: `cargo build --workspace`
- Test: `cargo test --workspace`
- Lint: `cargo clippy --workspace -- -D warnings -W clippy::pedantic`

## Rules

- Rust edition 2024, stable toolchain 1.96.0 only — no nightly features.

- Never use .expect() or .unwrap() on Result/Option — use ? or explicit match/if let.

- Never use index slicing that can panic (e.g. &vec[i], &s[0..n]) — use .get() with explicit handling.

- All code must compile with clippy::pedantic at maximum sensitivity with zero warnings.

- Do not add #[allow(clippy::...)] suppression attributes in production code unless absolutely unavoidable.

- Keep domain types and port traits free of I/O framework dependencies (no tokio, no reqwest in domain module).

- All error types use thiserror; anyhow only in the CLI binary crate.

- Use conventional commits for all commit messages.

- Use Coraline (coraline_* MCP tools) for code navigation when exploring the codebase.

- Two workspace crates: accroitre (library, published) and accroitre-cli (binary, not published).

- The library crate is the only crate published to crates.io.

- The binary crate produces the `accro` CLI command.

- Use LazyLock instead of lazy_static or once_cell.

- Prefer async closures (async || {}) where appropriate — Rust 2024 edition feature.

- Use let chains (if let && let) where they improve readability — Rust 2024 edition feature.
